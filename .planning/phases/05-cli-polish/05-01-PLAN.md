  <action>
Create three command modules under src/cli/:

**src/cli/status.ts** — `anvil status` (CLI-02):
- Export a `statusCommand` that is a `new Command('status')`.
- Reads `.anvil/roadmap.json` for plan info (task count, task descriptions).
- Reads `.anvil/reports/wave-*-judges.json` files for wave progress and judge verdicts.
- Reads `.anvil/high-court-report.json` if it exists for final verdict.
- Displays a formatted summary:
  - Build state header: project name, timestamp of last report
  - Wave progress: for each wave report, show wave number, tasks merged/failed, judge verdicts (green checkmark for pass, red X for fail) using chalk
  - High Court verdict (if exists): colored by outcome (green=merge, yellow=human_required, red=abort)
  - If no .anvil/ artifacts found, print "No build data found. Run `anvil run` to start a build."
- Use `readdir` + `readFile` from `node:fs/promises`, `join` from `node:path`.
- Parse JSON files through Zod schemas (SubJudgeReportSchema, HighCourtReportSchema) for safety.
- Color: green for passed, red for failed, yellow for warnings (per CLUX-02).

**src/cli/cost.ts** — `anvil cost` (CLI-03):
- Export a `costCommand` that is a `new Command('cost')`.
- Reads `.anvil/cost-report.json`, parses through CostReportSchema.
- Reuse `formatCostSummary` from `src/cost/display.ts` for the main table.
- Add a `--by-wave` flag: when set, group entries by waveNumber and show per-wave subtotals.
- If no cost report found, print "No cost data found. Run `anvil run` to generate a cost report."

**src/cli/logs.ts** — `anvil logs` (CLI-04):
- Export a `logsCommand` that is a `new Command('logs')`.
- Option `--wave <n>`: filter to logs from a specific wave number.
- Option `--task <id>`: filter to logs mentioning a specific task ID.
- Option `--level <level>`: filter by log level (info, warn, error). Default: show all.
- Option `-n, --lines <n>`: number of lines to show (default 50, from tail).
- Reads `.anvil/logs/anvil.log` which is newline-delimited JSON (pino format).
- Parse each line as JSON, filter by criteria, format for display:
  - Timestamp (short format HH:MM:SS), level (colored), message, optional task/wave fields.
- If log file missing or empty, print "No logs found. Logs are generated during `anvil run`."

All three modules: use chalk for coloring, handle file-not-found gracefully (no crashes), export the Command instance.
