**Class ProgressDisplay:**
- Constructor takes `{ silent?: boolean }` for test mode (suppresses ora spinners).
- Private `spinner: Ora | null` field.

**Wave lifecycle methods (CLUX-01):**
- `waveStart(waveNumber: number, totalWaves: number, taskCount: number): void` — Start an ora spinner with text `Wave {n}/{total} — {taskCount} tasks`. Use chalk.cyan for the wave header.
- `taskStart(waveNumber: number, taskId: string): void` — Update spinner text to show current task. Format: `Wave {n} | {taskId}...`
- `taskComplete(waveNumber: number, taskId: string, filesWritten: number): void` — Print (not spinner) a green checkmark line: `  ✓ {taskId} ({filesWritten} files)`. Use `spinner.clear()` then `console.log()` then `spinner.start()` to print between spinner frames.
- `taskFailed(waveNumber: number, taskId: string, error: string): void` — Print red X: `  ✗ {taskId} — {error}`.
- `judgeResult(check: { name: string; passed: boolean; message?: string }): void` — Print green `  ✓ Judge: {name}` or red `  ✗ Judge: {name} — {message}`.
- `waveComplete(waveNumber: number, mergedCount: number): void` — Stop spinner with succeed, print green summary line.
- `waveHalted(waveNumber: number, reasons: string[]): void` — Stop spinner with fail, print yellow halt message.

**Color helpers (CLUX-02):**
- `passed(text: string): string` — returns `chalk.green(text)`
- `warned(text: string): string` — returns `chalk.yellow(text)`
- `failed(text: string): string` — returns `chalk.red(text)`
- `info(text: string): string` — returns `chalk.blue(text)`

**Completion banner (CLUX-03):**
- `printCompletionSummary(result: { success: boolean; taskCount: number; waveCount: number; verdict?: string; totalCost?: number; failedAt?: number; failedTasks?: string[] }): void`
- On success with merge verdict:
  ```
  ┌─────────────────────────────────────┐
  │  ✓ Build Complete                   │
  │                                     │
  │  Tasks: {n} across {w} wave(s)      │
  │  Verdict: merge                     │
  │  Cost: ${total}                     │
  │                                     │
  │  Next steps:                        │
  │    anvil status   — review build    │
  │    anvil cost     — cost breakdown  │
  │    anvil logs     — detailed logs   │
  │    git push       — ship it         │
  └─────────────────────────────────────┘
  ```
  Use chalk.green for the border and header.
- On failure: Same box format but with red border, show failed wave number and failed task IDs. Next steps suggest `anvil logs --wave {n}` and `anvil status`.
- On abort/human_required: Yellow border, show verdict and suggest checking `.anvil/high-court-report.json`.
