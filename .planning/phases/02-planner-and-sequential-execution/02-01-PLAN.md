2. Create `src/core/topological-sort.ts` implementing Kahn's algorithm:
   - Export `function topologicalSort(tasks: Task[]): Task[]`
   - Import `Task` from `../schemas/plan.js`
   - Build adjacency list from `task.dependsOn` fields
   - Compute in-degree for each task
   - Process queue of zero-in-degree tasks
   - If `sorted.length !== tasks.length`, throw `Error('Dependency cycle detected among tasks: ...')` listing the remaining task IDs joined by comma-space
   - Return the sorted Task[] array
   - Also export `function validateDependencyRefs(tasks: Task[]): string[]` that returns error strings for any `dependsOn` ID that doesn't match any task's `id`

3. Extend `src/core/validator.ts`:
   - Add import for `Task` from `../schemas/plan.js`
   - Add and export `function detectWriteOverlaps(tasks: Task[]): Array<{ taskA: string; taskB: string; overlappingFiles: string[] }>`
     - For each pair of tasks (i, j where j > i), compute intersection of `tasks[i].writes` and `tasks[j].writes`
     - If intersection is non-empty, push `{ taskA: tasks[i].id, taskB: tasks[j].id, overlappingFiles: [...intersection] }`
     - Return the array (empty array = no overlaps)
   - Add and export `function validatePlanFull(data: unknown): ValidationResult` that:
     - Calls existing `validatePlan(data)` first
     - If valid, runs `detectWriteOverlaps(plan.tasks)` and `validateDependencyRefs(plan.tasks)` (imported from topological-sort.ts)
     - If overlaps found, returns `{ valid: false, errors: ['Write overlap: tasks X and Y both write to: file1, file2'] }`
     - If bad dependency refs found, returns `{ valid: false, errors: [...ref errors] }`
     - Otherwise returns the original valid result
   - Keep existing `validatePlan` function unchanged for backward compatibility

4. Update `src/index.ts` barrel to add:

2. Create `src/stations/planner.ts`:
   - Import `Anthropic` from `@anthropic-ai/sdk`
   - Import `zodOutputFormat` from `@anthropic-ai/sdk/helpers/zod`
   - Import `PlanSchema, type Plan` from `../schemas/plan.js`
   - Import `type AnvilConfig` from `../schemas/config.js`
   - Import `detectWriteOverlaps` from `../core/validator.js`
   - Import `validateDependencyRefs` from `../core/topological-sort.js`
   - Import `PLANNER_SYSTEM_PROMPT` from `../prompts/planner-system.js`
   - Export `async function generatePlan(spec: string, config: AnvilConfig, options?: { client?: Anthropic; maxRetries?: number }): Promise<Plan>`
     - Create Anthropic client: `options?.client ?? new Anthropic()`
     - Call `client.messages.parse()` with:
       - `model: config.model`
       - `max_tokens: 16384`
       - `system: PLANNER_SYSTEM_PROMPT`
       - `messages: [{ role: 'user', content: spec }]`
       - `output_config: { format: zodOutputFormat(PlanSchema) }`
     - Extract plan from `response.parsed_output`
     - If `parsed_output` is null/undefined, throw `Error('Planner produced no output')`
     - Run overlap detection: `detectWriteOverlaps(plan.tasks)`
     - If overlaps found AND retries remaining (default maxRetries=3):
       - Build a re-prompt message describing the overlaps: `"Your plan has write overlaps that must be fixed:\n${overlaps.map(o => ...).join('\n')}\nPlease regenerate the plan with no overlapping writes."`
       - Recursively call with decremented retry count and the overlap feedback appended to messages
     - If overlaps found AND no retries remaining, throw `Error('Planner failed to resolve write overlaps after 3 attempts')`
     - Run `validateDependencyRefs(plan.tasks)` — if errors, throw `Error('Invalid dependency references: ...')`
     - Return the validated plan
