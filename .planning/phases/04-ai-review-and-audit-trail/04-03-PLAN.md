      1. Generate project documentation from the provided build context
      2. For README.md: project name, description, installation, usage, project structure, key features

    **Step 2: Create `src/stations/librarian.ts`:**

    Export `runLibrarian` with signature:
    ```typescript
    export async function runLibrarian(
      projectDir: string,
      plan: Plan,
      highCourtReport: HighCourtReport,
      config: AnvilConfig,
      options?: { client?: Anthropic },
    ): Promise<{ readmePath: string; architecturePath: string }>
    ```

    Implementation:
    1. Create client: `options?.client ?? new Anthropic()`
    2. Gather context:
       - Directory tree: use `readdir` recursive to build file list (filter out node_modules, .git, .anvil, dist)
       - Read `package.json` if exists
       - High Court report reasoning and concerns
       - Plan spec and task descriptions
    3. Build user message for README with: project name (from config), plan spec, file tree, package.json contents, High Court reasoning
    4. Call `client.messages.create()` with model, max_tokens: 4096, system prompt, user message
    5. Extract text content from response (filter for text blocks, join content)
    6. Write to `join(projectDir, 'README.md')`
    7. Build user message for ARCHITECTURE with: file tree, task descriptions with writes[], High Court invariantChecks, High Court reasoning
    8. Call `client.messages.create()` again
    9. Extract text, write to `join(projectDir, 'ARCHITECTURE.md')`
    10. Return `{ readmePath: join(projectDir, 'README.md'), architecturePath: join(projectDir, 'ARCHITECTURE.md') }`

    Do NOT commit in this function — commit happens in the CLI wiring (Plan 04). This keeps the function pure and testable.

    **Step 3: Write unit tests** in `tests/unit/librarian.test.ts`:
    - Mock Anthropic client: `{ messages: { create: vi.fn().mockResolvedValue({ content: [{ type: 'text', text: '# Mock Doc' }], usage: { input_tokens: 100, output_tokens: 200 } }) } }`
    - Use temp directory with `mkdtemp` for projectDir
    - Create minimal package.json and a few dummy files in temp dir
    - All 7 behavior tests
